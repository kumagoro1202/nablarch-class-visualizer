# Nablarch クラス可視化ツール — 要件定義書

**バージョン**: 1.0  
**作成日**: 2026-05-20  
**参照元**: 技術選定調査およびクラス数計測結果

---

## 1. 背景

Nablarch フレームワークは118リポジトリ（本番 Java ソースを持つもの92件）に分散して配布されています。2026-05-19時点で、92リポジトリ全体に **2,681件の型宣言**（クラス・インタフェース・列挙型・アノテーション・レコード）が存在します。継承・実装・利用といったクラス間の関係を把握することは、オンボーディング・アーキテクチャレビュー・コードリーディングにおいて不可欠です。

この規模では単一の静的ダイアグラムは現実的ではありません。本ツールは **インタラクティブな探索ビュー**を提供し、高レベルの俯瞰から特定の関係を段階的に深掘りできるようにします。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|----------|
| US-01 | 開発者として、Nablarch の全クラスを1つのキャンバスで確認し、アーキテクチャの全体像を把握したい。 |
| US-02 | 開発者として、クラスを選択して関連クラスを段階的に展開し、視覚的ノイズなしに特定の継承チェーンを追跡したい。 |
| US-03 | 開発者として、関係性タイプでフィルタリングし、継承パターンと利用パターンを個別に確認したい。 |
| US-04 | 開発者として、アーティファクト（JAR）でフィルタリングし、`nablarch-fw-web` などの特定モジュールに集中したい。 |
| US-05 | 開発者として、クラス名で検索してハイライト・中央表示することで、対象クラスを素早く見つけたい。 |
| US-06 | 開発者として、参照したい Nablarch バージョンを選択し、そのバージョンのクラス関係を確認したい。 |
| US-07 | 開発者として、新バージョンの解析を手動ステップなしに起動し、最新データを取得したい。 |

## 3. 機能要件

### 3.1 探索ビュー

#### 3.1.1 初期俯瞰

- **REQ-V01**: ロード時に全クラスを1つのキャンバスに表示し、パッケージグループまたはアーティファクトグループで簡略化したレイアウトを用いる。
- **REQ-V02**: 俯瞰スケールでは、2,681件以上のノードの視認性を維持するため、個別クラスのラベルを非表示または省略してよい。
- **REQ-V03**: 各ノードはアーティファクト（JAR）ごとに色分けする。カラーレジェンドを常時表示する。
- **REQ-V04**: 初期レイアウトは、モダンなノートPCで **10秒以内** に描画・インタラクション可能な状態になること。

#### 3.1.2 クラス選択とN段階展開

- **REQ-V05**: ノードをクリックすると「フォーカスクラス」として選択され、直接関連するクラスが展開される（N=1）。
- **REQ-V06**: 展開コントロール:
  - **+1 レベル**: 現在のフォーカスクラスからさらに1ホップ展開
  - **−1 レベル**: 最外リングを折り畳む
  - **全展開**: フォーカスクラスから推移的に到達可能な全クラスを表示
  - **リセット**: 初期俯瞰に戻る
- **REQ-V07**: 展開されていないノードはキャンバスから削除せず、ユーザー設定に応じて薄暗く表示または非表示にする。

#### 3.1.3 関係性タイプフィルタ

- **REQ-V08**: 表示するエッジを関係性タイプでフィルタリングできる。最低4タイプをサポートする:
  1. **継承 / 実装** (`extends` / `implements`)
  2. **利用** (`import` / フィールド参照 / メソッド呼び出し)
  3. **包含** (内部クラス / ネストクラス)
  4. **依存** (コンストラクタ引数型 / フィールド型)
- **REQ-V09**: 複数の関係性タイプを同時に有効化できる。
- **REQ-V10**: フィルタ変更時、全体リロードなしにグラフを更新する。

#### 3.1.4 アーティファクトフィルタ

- **REQ-V11**: 1つ以上のアーティファクト（例: `nablarch-fw-batch`, `nablarch-fw-web`, `nablarch-core`）を選択して表示ノードを絞り込める。
- **REQ-V12**: アーティファクト選択はマルチセレクトコントロールとする。デフォルトは「全て」。
- **REQ-V13**: フィルタリストは解析データから動的に生成する（ハードコードしない）。

#### 3.1.5 パッケージ階層フィルタ

- **REQ-V14**: パッケージプレフィックス（例: `nablarch.fw.*`）でノードをフィルタリングできる。
- **REQ-V15**: 部分一致またはグロブパターンをサポートするテキスト入力で実装してよい。

#### 3.1.6 検索とハイライト

- **REQ-V16**: 検索ボックスでクラス名またはパッケージ名を入力できる（部分一致対応）。
- **REQ-V17**: マッチしたノードをハイライトし、最初のマッチに自動的にカメラを移動する。
- **REQ-V18**: 複数マッチがある場合、ユーザーがそれらを巡回できる。
- **REQ-V19**: 不要な再描画を避けるため、検索入力をデバウンス処理する（200ms以上）。

#### 3.1.7 ナビゲーション

- **REQ-V20**: マウスホイールズームをサポートし、最小・最大ズームレベルを設定可能にする。
- **REQ-V21**: 空白スペースのクリック＆ドラッグによるキャンバスパンをサポートする。
- **REQ-V22**: ノードのドラッグによる手動配置変更をサポートする。

### 3.2 バージョン管理

#### 3.2.1 バージョン選択

- **REQ-M01**: ドロップダウンから解析済みの Nablarch バージョンを選択できる。
- **REQ-M02**: 「バージョン」は Nablarch リリースタグ（例: `v5.4.0`）または Git コミット SHA で定義する。
- **REQ-M03**: UIは現在選択中のバージョンとその解析タイムスタンプを表示する。

#### 3.2.2 解析実行

- **REQ-M04**: 「このバージョンを解析」ボタンを押すと、指定バージョンの解析パイプラインを起動する。
- **REQ-M05**: 解析中は進捗インジケーターで現在の状態を表示する:
  - 状態遷移: `キュー待ち` → `クローン中` → `解析中` → `JSON生成中` → `完了` / `失敗`
  - 推定所要時間を表示する（目標: 全解析で15分以内）
- **REQ-M06**: 解析失敗時はエラーメッセージと部分的なログ抜粋を表示する。
- **REQ-M07**: 完了後、新バージョンがバージョンセレクタに追加される。

#### 3.2.3 バージョン切り替え

- **REQ-M08**: 解析済みバージョン間を再解析なしに切り替えられる。
- **REQ-M09**: バージョン切り替えはグラフデータを再読み込みするが、可能な範囲でUIの状態（ズームレベル・有効フィルタ）を維持する。

#### 3.2.4 解析結果の永続化

- **REQ-M10**: 解析出力は `data/versions/{version}/` 配下に JSON ファイルとして保存する（データスキーマ参照）。
- **REQ-M11**: 保存された結果はブラウザセッションおよびアプリケーション再起動をまたいで保持される。
- **REQ-M12**: 解析済みバージョンの一覧（解析日・クラス数・所要時間を含むメタデータ）を `data/versions/index.json` に管理する。

## 4. 非機能要件

| ID | カテゴリ | 要件 |
|----|---------|------|
| NFR-01 | パフォーマンス | モダンなノートPC（Chrome/Firefox）で2,681ノードの初期描画を10秒以内に完了すること。 |
| NFR-02 | パフォーマンス | 関係性フィルタの切り替えを1秒以内に反映すること。 |
| NFR-03 | パフォーマンス | クラス検索結果を500ms以内に表示すること。 |
| NFR-04 | スケーラビリティ | クラス数増加を見越し、アーキテクチャ変更なしで最大5,000ノードに対応できる設計とする。 |
| NFR-05 | ブラウザ対応 | Chrome（最新版）・Firefox（最新版）・Edge（最新版）。Safari はベストエフォート。 |
| NFR-06 | オフライン | 初回ロード後はネットワーク接続なしにレンダリングフロントエンドが動作すること（データはローカルJSON）。 |
| NFR-07 | ポータビリティ | macOS・Windows（WSL2）・Linux でOS固有の変更なしに動作すること。 |

---

## 5. スコープ外

- リアルタイムコード解析（解析は常に事前ステップとして実行し、結果はJSONとして永続化する）
- Nablarch ソースコードの変更
- メソッドレベル・フィールドレベルの依存関係可視化（本バージョンはクラスレベルのみ）
- IDEツールとの統合
- モバイル・スマートフォン対応（デスクトップおよびタブレットのみ）

## 6. 用語集

| 用語 | 定義 |
|------|------|
| **ノード** | 単一の型宣言（クラス・インタフェース・列挙型・アノテーション・レコード） |
| **エッジ** | 2つのノード間の有向関係 |
| **アーティファクト** | Maven JAR アーティファクト（例: `nablarch-fw-web`） |
| **バージョン** | 解析対象とする Nablarch リリースタグまたはコミット SHA |
| **フォーカスクラス** | N段階展開の中心として現在選択されているノード |
| **俯瞰モード** | 展開フィルタなしに全ノードを表示する初期キャンバス状態 |
